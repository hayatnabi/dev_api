API Documentation for CRUD Operations

1. GET

  • Description: Fetches data from the server.
  • Example Usage: Retrieve a list of resources or a specific resource.
  • Endpoint: http://localhost:3000/posts
  • Response: JSON object containing the requested data.

2. POST

  • Description: Creates a new resource on the server.
  • Example Usage: Submit data to create a new entry.
  • Endpoint: http://localhost:3000/posts
  • Request Body: JSON object containing the data for the new resource.
  • Response: JSON object with the created resource and status.

3. PUT

  • Description: Updates an existing resource on the server.
  • Example Usage: Modify data of an existing entry.
  • Endpoint: http://localhost:3000/posts/:id
  • Request Body: JSON object containing the updated data.
  • Response: JSON object with the updated resource and status.

4. DELETE

  • Description: Deletes an existing resource from the server.
  • Example Usage: Remove a specific resource.
  • Endpoint: http://localhost:3000/posts/:id
  • Response: Status message indicating success or failure.
